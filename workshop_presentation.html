BAGIAN 1: PEMBUKA Problem Framing

Server crash saat traffic naik, biaya mahal saat traffic sepi.

"Developer harusnya fokus nulis kode, bukan ribet urus kabel server."

BAGIAN 2: KONSEP DASAR Definisi Cloud Computing

Apa itu Cloud Computing?

"Penyampaian layanan komputasi (server, storage, database) melalui Internet ('the cloud') untuk menawarkan inovasi cepat, sumber daya fleksibel, dan skala ekonomi."

— NIST Definition

1. Abstraksi

Fisik hardware disembunyikan. Anda melihat "Virtual Resource".

2. Elastisitas

Kapasitas bisa membesar dan mengecil sesuai kebutuhan seketika.

3. Managed Services

Provider yang mengelola maintenance, security, dan power.

BAGIAN 2: KONSEP DASAR Prinsip Dasar

Karakteristik Cloud Computing

Tiga pilar fundamental yang membedakan cloud computing dari infrastructure tradisional.

On-Demand Self-Service

Klik, dan server siap dalam hitungan detik. Tanpa admin manual. Provisioning otomatis tanpa harus menghubungi tim IT.

Scalability

Scale UP (tambah power) atau Scale OUT (tambah jumlah unit) otomatis. Kapasitas menyesuaikan dengan beban kerja secara real-time.

Automation

Infrastructure as Code (IaC). Server diatur lewat script, bukan klik manual. Reproducible dan version-controlled.

BAGIAN 3: EVOLUSI Evolusi Arsitektur

Perjalanan Arsitektur Aplikasi

Bagaimana cara membangun aplikasi telah berevolusi dari satu blok besar menjadi fungsi-fungsi kecil yang independen.

Monolithic

Semua komponen aplikasi (UI, logic, database) digabung menjadi satu kesatuan besar.

  • ✓ Simple to develop awalnya
  • ✗ Deploy satu = deploy semua
  • ✗ Scaling seluruh app, bukan per-fitur

Microservices

Aplikasi dipecah menjadi service-service kecil yang berkomunikasi via API.

  • ✓ Independent deployment
  • ✓ Tech stack bisa berbeda
  • ✗ Kompleksitas DevOps tinggi

Serverless

Fungsi-fungsi kecil yang dijalankan cloud provider. Tidak perlu manage server sama sekali.

  • ✓ Zero infrastructure management
  • ✓ Auto-scale per function
  • ✓ Pay only when code runs

BAGIAN 4: SERVERLESS Core Concepts

Tiga Prinsip Serverless

Konsep inti yang mendefinisikan paradigma serverless computing.

1. Stateless (Ephemeral)

Fungsi dibuat, dijalankan, lalu dihancurkan. Tidak ada data yang disimpan di memori server. State wajib di Database. Setiap eksekusi independen dan tidak tahu tentang eksekusi sebelumnya.

2. Event-Driven

Hanya bangun jika ada trigger (HTTP Request, DB Change, Timer). Tidak ada server yang standby idle. Fungsi "tidur" saat tidak digunakan, hemat resource dan biaya.

3. Pay-per-Use

Bayar per milidetik eksekusi. Tidak ada traffic = tagihan Rp 0. Model billing yang sangat cost-efficient untuk aplikasi dengan traffic tidak konstan.

BAGIAN 4: SERVERLESS Event-Driven Flow

Bagaimana Serverless Bekerja?

Serverless itu reaktif. Fungsi tidak berjalan terus-menerus, melainkan "tidur" sampai ada event yang menyentuhnya.

"Tidak ada traffic = tidak ada biaya. Fungsi hanya hidup saat dibutuhkan."

Contoh Alur:

1
Event Source: User upload foto ke S3 bucket
2
Function Triggered: Lambda otomatis resize gambar
3
Response/Action: Simpan hasil ke DB, kirim notifikasi
BAGIAN 4: SERVERLESS Trade-offs

Kelebihan vs Keterbatasan Serverless

Seperti teknologi lainnya, serverless memiliki trade-off. Penting untuk memahami kapan serverless cocok dan kapan tidak.

Kelebihan

Cost Efficiency

Bayar hanya saat code berjalan. Ideal untuk traffic yang fluktuatif atau aplikasi dengan usage tidak konstan.

Auto-Scaling

Dari 0 ke ribuan request per detik secara otomatis. Tidak perlu konfigurasi manual.

Faster Time-to-Market

Fokus menulis business logic, bukan setup server. Development cycle lebih cepat.

Keterbatasan

Cold Start Latency

Fungsi yang lama tidak dipanggil butuh waktu "bangun" (~100ms-3s). Bisa jadi masalah untuk real-time apps.

Vendor Lock-in

Kode bergantung pada API spesifik provider (AWS, GCP). Migrasi antar cloud tidak mudah.

Debugging Complexity

Sistem terdistribusi lebih sulit di-debug. Butuh alat observability khusus.

Saatnya Praktik!

Lab 1: Netlify Functions

Langkah Pengerjaan

  1. Sign Up Netlify: Daftar di netlify.com menggunakan akun GitHub Anda (gratis, tanpa kartu kredit).
  2. Fork Repository: Fork starter repo yang disediakan ke akun GitHub Anda.
  3. Deploy: Di Netlify Dashboard → Add New Site → Import from Git → Pilih repo yang di-fork.
  4. Write Function: Edit file di folder netlify/functions/hello.js, push ke GitHub.
  5. Test: Akses URL function: your-site.netlify.app/.netlify/functions/hello
Lab 2: Vercel

Langkah Pengerjaan

  1. Prepare Repo: Fork/Clone repository starter code Next.js yang disediakan ke akun GitHub Anda.
  2. Vercel Dashboard: Login ke Vercel → Klik "Add New Project".
  3. Import Git: Pilih repository yang baru saja Anda fork.
  4. Deploy: Biarkan setting default, klik "Deploy". Tunggu ~1 menit.
  5. CI/CD Demo: Edit file page.jsx di lokal, commit & push, lalu lihat Vercel me-rebuild otomatis.
Lab 3: Firebase

Langkah Pengerjaan

  1. Create Project: Buka Firebase Console → Buat project baru (matikan Analytics).
  2. Init CLI: Di terminal lokal, jalankan firebase login lalu firebase init functions.
  3. Write Function: Buka file functions/index.js dan buat fungsi trigger onDocumentCreated.
  4. Deploy: Jalankan perintah firebase deploy --only functions.
  5. Trigger Event: Buat dokumen baru di Firestore Database dan lihat Logs untuk verifikasi fungsi berjalan.
Lab 4: Integration

Langkah Pengerjaan

  1. Open Test Page: Buka file integration-test.html (disediakan) di browser.
  2. Input URLs: Masukkan URL API Gateway (dari Lab 1) dan URL Vercel (dari Lab 2).
  3. Test CORS: Klik tombol "Test Endpoint". Jika merah, Anda harus menambahkan Header CORS di Lambda.
  4. Fix CORS: Update kode Lambda untuk return header Access-Control-Allow-Origin: *.
  5. Validasi: Test ulang hingga muncul status hijau (Success 200).
